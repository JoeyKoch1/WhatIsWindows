Memory Management Functions

Virtual memory, heaps, and memory-mapped files

Contents

  • Overview
  • VirtualAlloc
  • Heap Functions
  • Related Components

Overview

Windows provides multiple memory allocation mechanisms for different use cases:

  • Virtual Memory - Low-level page allocation (VirtualAlloc)
  • Heap Memory - Flexible small allocations (HeapAlloc)
  • Memory-Mapped Files - File-backed memory regions
  • Process Heap - Default heap (GetProcessHeap)

VirtualAlloc

VirtualAlloc reserves or commits pages in virtual address space. It's the lowest-level allocation function.

c
// Reserve and commit 1MB of memory
LPVOID pMem = VirtualAlloc(
  NULL,
  1024 * 1024,
  MEM_COMMIT | MEM_RESERVE,
  PAGE_READWRITE
);

if (pMem) {
  // Use memory
  memset(pMem, 0, 1024 * 1024);
  
  // Free when done
  VirtualFree(pMem, 0, MEM_RELEASE);
}

Heap Functions

Heap functions provide faster allocation for small objects:

c
// Create a private heap
HANDLE hHeap = HeapCreate(0, 0, 0);

// Allocate from heap
LPVOID pBlock = HeapAlloc(hHeap, 0, 1024);

// Free the block
HeapFree(hHeap, 0, pBlock);

// Destroy heap
HeapDestroy(hHeap);
