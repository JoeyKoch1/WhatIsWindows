Windows Memory Model

Virtual memory management, page tables, and address space layout

Contents

  • Overview
  • Address Space
  • Page Tables
  • VAD Tree
  • Related Components

Overview

Windows uses a flat virtual memory model where each process has its own virtual address space. The Memory Manager translates virtual addresses to physical addresses using page tables managed by the hardware MMU (Memory Management Unit).

Fun Fact: Windows 95/98 used a shared 4GB address space where all processes could see each other's memory! Modern Windows gives each process its own isolated 256 TB address space on x64.

Address Space Layout x64

On x64 Windows, each process has a 256 TB virtual address space split between user and kernel mode:

  • User Mode: 0x00000000`00000000 to 0x00007FFF`FFFFFFFF (128 TB)
  • Kernel Mode: 0xFFFF8000`00000000 to 0xFFFFFFFF`FFFFFFFF (128 TB)

Memory States

StateDescriptionPhysical Memory
FreeAvailable for allocationNot allocated
ReservedAddress range reserved but not committedNot allocated
CommittedBacked by physical memory or pagefileAllocated on demand
Note: Committing memory doesn't immediately allocate physical RAM. Windows uses demand paging - physical pages are only allocated when actually accessed (page fault).

Page Tables

x64 uses 4-level page translation to convert virtual addresses to physical addresses:

PML4Page Map Level 4Bits 39-47512 entries
PDPPage Directory PointerBits 30-38512 entries

Virtual Address Format (x64):
+--------+--------+--------+--------+---------+
|  PML4  |  PDP   |   PD   |   PT   | Offset  |
| 9 bits | 9 bits | 9 bits | 9 bits | 12 bits |
+--------+--------+--------+--------+---------+
  47-39    38-30    29-21    20-12     11-0

Total: 48 bits addressed (256 TB)
Each level: 512 entries * 8 bytes = 4KB per table

Translation Example: When accessing virtual address 0x00007FFF`12345678:

  1. CPU reads CR3 register to find PML4 base address
  2. Uses bits 39-47 to index into PML4 → gets PDP address
  3. Uses bits 30-38 to index into PDP → gets PD address
  4. Uses bits 21-29 to index into PD → gets PT address
  5. Uses bits 12-20 to index into PT → gets physical page frame
  6. Adds offset (bits 0-11) to get final physical address

VAD Tree

The Virtual Address Descriptor (VAD) tree tracks all memory allocations in each process. It's implemented as a balanced binary tree (AVL tree) stored in the EPROCESS structure.

VAD Entry Information

  • Starting and ending virtual addresses
  • Memory protection (PAGE_READWRITE, PAGE_EXECUTE, etc.)
  • Memory type (private, mapped, image)
  • Commit charge
  • Associated file mapping (if any)
Kernel Debugging Tip: Use !vad command in WinDbg to dump the VAD tree for any process. This shows all memory regions and their attributes.
