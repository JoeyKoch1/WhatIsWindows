Windows NT Architecture

System design, privilege levels, and component organization

Overview

Windows NT uses a hybrid kernel architecture that combines microkernel and monolithic kernel design principles. The operating system is divided into two main execution modes: user mode and kernel mode, providing hardware protection and system stability.

+--------------------------------------------------+ | User Mode Applications | | +------------+ +------------+ +------------+ | | | Win32 App | | .NET App | | UWP App | | | +------------+ +------------+ +------------+ | | | | | | | +------------------------------------------------+ | | System DLLs (User Mode) | | | kernel32 | user32 | ntdll | advapi32 | | +------------------------------------------------+ | | SYSCALL | +--------------------------------------------------+ | Kernel Mode (Ring 0 / CPL 0) | | +----------------------------------------------+| | | Windows Executive (ntoskrnl) || | | I/O | Memory | Process | Security | Config || | +----------------------------------------------+| | | NT Kernel || | | Scheduler | Dispatcher | Synchronization || | +----------------------------------------------+| | | Hardware Abstraction Layer || | +----------------------------------------------+| +--------------------------------------------------+ | Physical Hardware | +--------------------------------------------------+

Privilege Levels

Windows uses x86/x64 hardware privilege levels (rings) to protect the operating system from user applications:

RingNameUsage in Windows
Ring 0Kernel ModeOS kernel, device drivers, full hardware access
Ring 1-2UnusedNot used by Windows
Ring 3User ModeApplications, limited API access

User Mode

User mode code runs with restricted privileges and cannot directly access hardware or kernel memory. Components include:

  • User Applications - Win32, .NET, UWP applications
  • System DLLs - kernel32.dll, user32.dll, ntdll.dll, etc.
  • Environment Subsystems - Win32, POSIX, OS/2 (legacy)
  • User-mode services - svchost.exe processes

Kernel Mode

Kernel mode has unrestricted access to system hardware and memory:

  • Executive - I/O Manager, Memory Manager, Process Manager, Security Reference Monitor
  • Kernel - Thread scheduler, dispatcher, synchronization, trap handling
  • Device Drivers - Hardware-specific kernel-mode drivers
  • HAL - Hardware Abstraction Layer for platform independence
