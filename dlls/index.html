System DLLs

Core Windows Dynamic Link Libraries and their exported functions

Windows system DLLs form the foundation of the Windows API. Each DLL provides a specific set of functionality, from low-level kernel access to high-level user interface components.

  • ntdll.dll

    Native API

    The NT Layer DLL - lowest-level user-mode interface to the Windows NT kernel. Exports Native API functions (Nt*), runtime library functions (Rtl*), and loader functions (Ldr*). Every Windows process loads ntdll.dll automatically.

  • kernel32.dll

    Win32 API

    Base Windows API library providing process and thread management, memory management, file I/O, and error handling. Most kernel32 functions are wrappers around ntdll.dll Native API calls.

  • user32.dll

    User Interface

    Windows user interface library. Manages window creation, message loops, input handling, menus, dialogs, and controls. Interfaces with win32k.sys in kernel mode for GUI operations.

  • advapi32.dll

    Advanced API

    Advanced Windows API for registry access, security functions, service control manager, event logging, and cryptographic operations. Provides access control and authentication functionality.

  • gdi32.dll

    Graphics

    Graphics Device Interface library for 2D graphics, font rendering, drawing operations, and device contexts. Handles GDI objects like pens, brushes, fonts, and bitmaps.

  • msvcrt.dll

    C Runtime

    Microsoft Visual C Runtime library. Implements standard C library functions including string manipulation, memory operations, mathematical functions, and I/O streams.
