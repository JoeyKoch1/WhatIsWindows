gdi32.dll

Graphics Device Interface - 2D Graphics and Font Rendering

Contents

  • Overview
  • GDI Objects
  • Device Contexts
  • Related Components

Overview

gdi32.dll implements the Graphics Device Interface (GDI), Windows' 2D graphics API. It provides functions for drawing lines, curves, text, and bitmaps, as well as managing graphics objects like pens, brushes, fonts, and regions.

GDI functions translate high-level drawing commands into device-specific operations, abstracting the underlying hardware. Modern Windows applications increasingly use Direct2D/DirectWrite, but GDI remains essential for compatibility.

GDI Objects

GDI uses several object types for drawing operations:

  • HPEN - Pen for drawing lines
  • HBRUSH - Brush for filling areas
  • HFONT - Font for text rendering
  • HBITMAP - Bitmap for image data
  • HRGN - Region for complex shapes
  • HPALETTE - Color palette

Device Contexts

A Device Context (DC) is a structure that defines a set of graphic objects and their attributes. All GDI drawing operations require a DC handle.

c
HDC hdc = GetDC(hwnd);

HPEN hPen = CreatePen(PS_SOLID, 2, RGB(255, 0, 0));
HBRUSH hBrush = CreateSolidBrush(RGB(0, 255, 0));

SelectObject(hdc, hPen);
SelectObject(hdc, hBrush);

Rectangle(hdc, 50, 50, 200, 200);

DeleteObject(hPen);
DeleteObject(hBrush);
ReleaseDC(hwnd, hdc);
