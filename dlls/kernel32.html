kernel32.dll

Base Win32 API Library - Process, Memory, and I/O Management

Contents

  • Overview
  • Architecture
  • Key Functions
  • VirtualAlloc
  • CreateProcess
  • Related Components

Overview

kernel32.dll is the primary Win32 API library that provides the foundation for Windows application development. It exports functions for memory management, process and thread creation, file I/O, synchronization, and error handling. Almost every Windows application links against kernel32.dll.

Most kernel32.dll functions are wrapper functions that call down to the Native API exported by ntdll.dll. This abstraction layer provides a stable, documented API while allowing Microsoft to change the underlying Native API between Windows versions.

Architecture Note

On Windows NT systems, kernel32.dll is a thin wrapper around ntdll.dll. On Windows 9x systems, kernel32.dll contained the actual implementation. Modern Windows maintains this abstraction for backward compatibility.

Architecture

kernel32.dll sits between user applications and the Native API. When you call a kernel32 function, it typically performs parameter validation, translates Win32 types to Native API types, and invokes the corresponding Nt* function.

Function Categories

  • Memory Management - VirtualAlloc, HeapCreate, GlobalAlloc
  • Process Management - CreateProcess, OpenProcess, ExitProcess
  • Thread Management - CreateThread, SuspendThread, ExitThread
  • File I/O - CreateFile, ReadFile, WriteFile, SetFilePointer
  • Synchronization - CreateMutex, CreateEvent, WaitForSingleObject
  • Error Handling - GetLastError, SetLastError, FormatMessage
  • Module Loading - LoadLibrary, GetProcAddress, FreeLibrary

VirtualAlloc

VirtualAlloc reserves or commits a region of pages in the virtual address space of the calling process. It is the primary memory allocation function for low-level memory management.

Function Signature

LPVOID VirtualAlloc(
  LPVOID lpAddress,
  SIZE_T dwSize,
  DWORD  flAllocationType,
  DWORD  flProtect
);

Parameters

ParameterTypeDescription
lpAddressLPVOIDDesired starting address (NULL for system-selected)
dwSizeSIZE_TSize of region in bytes (rounded to page boundary)
flAllocationTypeDWORDMEM_COMMIT, MEM_RESERVE, or MEM_RESET
flProtectDWORDPAGE_READWRITE, PAGE_EXECUTE, PAGE_NOACCESS, etc.

Example: Allocating and Freeing Memory

c 
#include 
#include 

int main() {
  SIZE_T size = 4096;
  
  LPVOID pMem = VirtualAlloc(
    NULL,
    size,
    MEM_COMMIT | MEM_RESERVE,
    PAGE_READWRITE
  );
  
  if (pMem == NULL) {
    printf("Allocation failed: %lu\n", GetLastError());
    return 1;
  }
  
  memset(pMem, 0, size);
  printf("Allocated %zu bytes at %p\n", size, pMem);
  
  VirtualFree(pMem, 0, MEM_RELEASE);
  printf("Memory freed\n");
  
  return 0;
}

CreateProcess

CreateProcess creates a new process and its primary thread. The new process runs in the security context of the calling process.

Function Signature

BOOL CreateProcessA(
  LPCSTR                lpApplicationName,
  LPSTR                 lpCommandLine,
  LPSECURITY_ATTRIBUTES lpProcessAttributes,
  LPSECURITY_ATTRIBUTES lpThreadAttributes,
  BOOL                  bInheritHandles,
  DWORD                 dwCreationFlags,
  LPVOID                lpEnvironment,
  LPCSTR                lpCurrentDirectory,
  LPSTARTUPINFOA        lpStartupInfo,
  LPPROCESS_INFORMATION lpProcessInformation
);

Example: Launching a Process

c 
#include 
#include 

int main() {
  STARTUPINFO si;
  PROCESS_INFORMATION pi;
  
  ZeroMemory(&si, sizeof(si));
  si.cb = sizeof(si);
  ZeroMemory(&pi, sizeof(pi));
  
  if (!CreateProcess(
    NULL,
    "C:\\Windows\\System32\\notepad.exe",
    NULL,
    NULL,
    FALSE,
    0,
    NULL,
    NULL,
    &si,
    &pi
  )) {
    printf("CreateProcess failed: %lu\n", GetLastError());
    return 1;
  }
  
  printf("Process ID: %lu\n", pi.dwProcessId);
  printf("Thread ID: %lu\n", pi.dwThreadId);
  
  WaitForSingleObject(pi.hProcess, INFINITE);
  
  CloseHandle(pi.hProcess);
  CloseHandle(pi.hThread);
  
  return 0;
}
