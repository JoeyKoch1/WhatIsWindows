Windows Subsystems

Kernel-mode subsystems that form the core of Windows NT

Windows NT kernel-mode is organized into layered subsystems, each responsible for specific operating system functionality. These subsystems work together to provide process management, memory management, I/O operations, and security.

  • Executive

    Core

    The Windows Executive (ntoskrnl.exe) contains the core kernel components: I/O Manager, Memory Manager, Process Manager, Security Reference Monitor, and Configuration Manager. Provides high-level kernel services to user mode.

  • Kernel

    Core

    The NT Kernel provides thread scheduling, interrupt dispatching, synchronization primitives, and trap handling. Runs at IRQL DISPATCH_LEVEL and below. Interfaces directly with the Hardware Abstraction Layer.

  • HAL (Hardware Abstraction Layer)

    Hardware

    HAL abstracts hardware differences from the kernel. Provides platform-specific implementations of interrupt controllers, timers, DMA, and I/O port access. hal.dll varies by hardware platform.

  • win32k.sys

    Graphics

    Kernel-mode portion of the Windows graphics subsystem. Handles window management, GDI operations, input processing, and desktop management. Called by user32.dll and gdi32.dll via system calls.

  • LSA (Local Security Authority)

    Security

    Local Security Authority Subsystem Service (lsass.exe) manages security policies, user authentication, access tokens, and audit logging. Interfaces with the Security Accounts Manager (SAM) database.
